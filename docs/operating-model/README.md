# Operating Model

Capture working agreements for team-plus-agent execution.

## Required startup read order (all agents)

1. Read `docs/STATUS.md` first to get current phase, active objective, and next actions.
2. Read this entire folder (`docs/operating-model/`) before implementation work.
3. Treat `docs/operating-model/` as mandatory policy, not optional guidance.

Status and execution navigation:
- `docs/STATUS.md` is the operational single source of truth for current phase, objective, and next actions.
- `docs/plans/ROADMAP.md` tracks milestone-level progress.
- `docs/plans/NEXT-ACTIONS.md` tracks the active execution queue.

## Documentation map (canonical locations)

- Current state now: `docs/STATUS.md`
